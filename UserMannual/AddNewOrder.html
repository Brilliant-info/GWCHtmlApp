Inward â€“ Inward List

To view the inward menu user needs to login into the web & click on the inward menu then the user is able to see all parts of inward like inward order, receiving, quality check as well as put away & user able to do the different activities by using different menus like document, communication.

The various parts of this dialog are:

  1. Inward Order Type: In Inward Order type system will display the different order types of inward like purchases order type, sales return, and Transfer in.        
  2. Add New: By using this 'ADD NEW' button user is able to add new purchase order.
  3. Inward Orders: In inward orders, the tab system will display all the ongoing inward orders.
  4. Receiving: In this tab system will display all the received orders.
  5. Quality Check: In the quality check tab system will display all orders which are passed from the Quality check process.
  6. Put away: In the Put away tab system will display all the orders which are passed from the put-away process.
  7. More: If the user clicks on the more drop-down system will show a menu of cancel order, by using the cancel menu user will able to cancel the order.
  8. Document: In the document tab user can attach the document related to the order.
  9. Communication: By using the communication tab user can transfer order-related notes to the concerned person. 
  10. Search Filter: By using the filter user is able to sort the order as per mentioned filter type of Inward order no, Reference No & order date.

 Inward â€“ Create Sales Order


For creating a new purchase order user need to click on the add a new button which is provided on the inward order list then the user is able to view the above-mentioned page for adding new order.

  The various parts of this dialog are:

  1. Inward Order No: After filling all details regarding the inward order & clicking on the final save button system will generate the order No. which will display on the inward list for the further process.
  2. Order Date: The order date by default is the current date.
  3. Expected Delivery Date: The term describes the expected delivery date for the purchase order.
  4. Reference No: It is a number from which the user can identify his/her order easily.
  5. Vendor: The vendor is a supplier to the warehouse after clicking on the search button user is able to see the vendor list.
  6. Remark: In the remark, the field user can fill a short description of the order.
  7. Created By: In created by field system will display by which user order is getting placed.
  8. SKU code: In this field, the system will display the SKU code which is used for placing the purchase order. Also user can search for a new SKU by using magnifying glass from the SKU list.
  9. SKU Name: After selecting SKU code from the list system will display SKU name.
  10. Description: After selecting SKU code from the list system will display the SKU description. If the description is available for the selected SKU.
  11. Requested quantity:  A quantity of SKU requested by the user.User need to enter the quantity.Same process needs to repeat for all SKUs.
  12. UOM: Unit of measure in which the system will measure the quantity in a particular format like Each=1.
  13. LOTTABLE: LOTTABLE is a field in which the user can add the extra parameters of SKU like expiry date. If the LOTTABLE is present for the SKU then the user needs to enter the LOTTABLE while order creation.
  14. Fill all details of the weight field & click on the action button then SKU will be added successfully in order.
  15. Click on the final save button.

After Filling all the fields of creating order click on the save button then the order no will be generated & the order will display on the inward list.
